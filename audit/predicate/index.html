Predicate Index v0 — Stage 3 re-render (post CTO findings)

Stage 3 update (2026-05-09): CTO reviewed the original Stage-2 audit and reported 6 findings. Verifying #6 (turnover semantic) surfaced a 7th: the Stage-1 phase-join was using the wrong key. This re-render incorporates all 5 fixed findings + the phase-join fix. See VERIFIED_FINDINGS for the full list.

Fixes applied since last render:
  • #1 y-axis lane inverted — L↔R agreement against analyst labels went from 17% to 83% after inversion.
  • #2 "midfield" = central lane, not middle third (per CTO clarification).
  • #3 carry definition tightened — `meaningful_carry` requires n_carry≥1 AND ≥5m forward progress, kills CB-bouncing false positives.
  • #6 turnover/shot-or-goal flags now window-bounded, not phase-inherited.
  • #7 phase-join was using OUR poss_idx as the SC phase index — they're unrelated. Now uses frame-overlap. 20/20 spot-checked windows were joined to the wrong phase pre-fix.
Deferred: Finding #5 (Q6 same-side switch — needs `crosses_central_lane` predicate).
INTERNAL AUDIT — not a public demo. v9c canonical demo, BEST_CHECKPOINT, /query/, and reranker_v0 unchanged.
